# NA Room — Testing Brief

Этот документ для тестировщика: как запустить, что проверить, на что обратить особое внимание.

## Стек

- **Бэкенд:** Go 1.21+, SQLite3
- **Фронтенд:** SvelteKit 5 (Node.js)
- **Крипто:** BTC (mempool.space API), LTC (BlockCypher API)
- **Чат:** WebSocket + E2E шифрование (X25519 + XSalsa20-Poly1305, `nacl.box`)
- **Аналитика:** GoatCounter (только публичные страницы, без cookies)

## Требования

```bash
go 1.21+
node 18+
npm
```

## Запуск локально (dev-режим)

```bash
git clone https://github.com/naroomer/naroom
cd naroom

# Запустить бэкенд (DEV_MODE автоматически подтверждает инвойсы — реальных денег не нужно)
./dev.sh

# В отдельном терминале — фронтенд
cd frontend && npm install && npm run dev
```

Бэкенд: `http://localhost:8080`
Фронтенд: `http://localhost:5173`

В `DEV_MODE=true`:
- Инвойсы подтверждаются автоматически (без реальной оплаты)
- Баланс кошелька не проверяется
- Wallet адрес принимается любой (без проверки по блокчейну)

## Автоматические тесты

```bash
# Полный прогон: сборка + unit + frontend + 35 E2E тестов
./scripts/selftest.sh

# Только unit-тесты
./scripts/selftest.sh --unit-only

# Только E2E (нужен node)
./scripts/selftest.sh --e2e-only

# Расширенный прогон (включает Playwright тест 026)
FRONTEND_URL=http://localhost:4173 bash scripts/selftest-full.sh
```

Последний результат: **35/35 PASS**

## Ручное тестирование (без реальных денег)

### Полный флоу (клиент + peer)

1. Открой `http://localhost:5173/board/moscow` (или любой город)
2. Нажми "Мне нужна помощь" → создай объявление
3. В отдельном браузере/инкогнито открой тот же борд
4. Найди объявление → нажми "Откликнуться" (войди как peer)
5. В первом браузере прими отклик
6. Оба получат ссылку на чат → проверь что чат работает
7. Закрой чат → оставь рейтинг

В DEV_MODE все инвойсы ($5 за объявление, $15 за чат) подтверждаются мгновенно.

### Что проверить вручную отдельно

**Языки (важно для России):**
- Переключи язык на RU — все строки должны быть переведены
- Борды: Москва, Алматы, Ереван — языки русский/английский
- Создай объявление с языком `["ru"]` — убедись что отображается корректно

**Приватность (критично):**
- Открой DevTools → Network → убедись что аналитика (gc.zgo.at) НЕ загружается на `/new`, `/helper`, `/chat/*`, `/listing/*`
- Аналитика должна работать только на `/`, `/how-it-works`, `/board/*`

**Москва — особое внимание:**
- `http://localhost:5173/board/moscow` — должны быть образцы объявлений
- Создай объявление с городом Москва, языком RU
- Peer из другого города (например Тбилиси) НЕ должен мочь откликнуться на московское объявление (region_lock)
- Проверь что интерфейс полностью на русском

**WebSocket / чат:**
- Открой чат в двух вкладках
- Отправь сообщение с одной стороны — должно появиться у другой
- Закрой вкладку одного участника — второй должен увидеть статус "собеседник отключился"
- После закрытия чата сообщения должны быть удалены

**Сессии:**
- Верифицируй кошелёк → скопируй session_token
- Подожди/перезагрузи → токен должен оставаться действительным (24 часа)
- Нельзя использовать токен клиента для действий peer и наоборот

## Что нельзя проверить без реальных денег

- Реальный прием BTC транзакций (mempool.space API)
- Реальный прием LTC транзакций (BlockCypher API)
- Подтверждение что адреса действительно приходят в реальный Trezor

## Известные ограничения

- **Tor:** на локальном запуске не тестируется, только на production (.onion)
- **Telegram боты:** требуют реальные токены из `.env` — в dev.sh не настроены
- **Курс крипты:** в DEV_MODE используется заглушка, реальные цены только в production

## Конфигурация production (VPS)

- Домен: `naroom.net`
- Бэкенд: Go бинарник `/opt/naroom/naroom`, systemd `naroom.service`
- Фронтенд: SvelteKit Node.js `/opt/naroom-web/`, systemd `naroom-web.service`
- БД: `/opt/naroom/data/naroom.db` (SQLite)
- Аналитика: GoatCounter `naroom.goatcounter.com`

## Архитектура безопасности (кратко)

- **Кошелёк** никогда не хранится в открытом виде — только `HMAC-SHA256(HASH_KEY, address)`
- **Wallet address** в БД зашифрован AES-256-GCM (ключ `WALLET_ENC_KEY`)
- **Сообщения чата** — E2E шифрование, ключи только у клиентов, сервер видит только зашифрованные блобы
- **Логи** не содержат IP, кошельки, токены, тела запросов
- **Инвойсы** привязаны к кошельку — чужой кошелёк оплатить не может
